# WMS 数据库表结构说明

数据库名：`wmsdatabase`  
字符集：`utf8mb4` / `utf8mb4_unicode_ci`  
建表脚本：`init.sql`  
用户种子（可选）：`seed_users.sql`  
完整初始数据：由 `WmsServer/init_db.py` 写入（用户、仓库 WH01、库位、商品等）

---

## 表一览（13 张）

| 分类 | 表名 | 作用简述 |
|------|------|----------|
| 主数据 | `users` | 系统用户与角色 |
| 主数据 | `product` | 商品（编码） |
| 主数据 | `warehouse` | 仓库 |
| 主数据 | `location` | 库位 |
| 库存 | `inventory` | 现存量 |
| 库存 | `inventory_transaction` | 库存变动流水 |
| 入库 | `inbound_order` | 入库单主表 |
| 入库 | `inbound_order_line` | 入库单明细 |
| 出库 | `outbound_order` | 出库单主表 |
| 出库 | `outbound_order_line` | 出库单明细 |
| 盘点 | `stocktake_order` | 盘点单主表 |
| 盘点 | `stocktake_order_line` | 盘点单明细 |
| 辅助 | `order_no_seq` | 单据号按日序号 |

---

## 关系概览

```
users ── 创建/审核/操作 ── 各类单据、库存流水

warehouse ── location ── inventory（按 仓库+库位+编码 存数量）
     │
product ──┬── inventory / 流水 / 各单据明细

inbound_order ── inbound_order_line     （入库加库存）
outbound_order ── outbound_order_line   （出库减库存）
stocktake_order ── stocktake_order_line （盘点调库存）

inventory_transaction ← 记录每次数量变动，关联 ref_type + ref_id
```

**库存铁律**：`inventory.quantity` 只能通过后端 `inventory_service` 修改；每次变动必须写入 `inventory_transaction`。

## 主数据

### 1. `users` — 用户

| 字段 | 说明 |
|------|------|
| `username` | 登录名，唯一 |
| `name` | 姓名 |
| `password` | 密码哈希（非明文） |
| `role` | `admin` / `warehouse_keeper` / `viewer` |
| `is_active` | 是否启用 |
| `last_login_at` | 最近登录时间 |

用于登录鉴权（JWT）。权限以 `WmsServer` 路由装饰器为准：

- `admin`：主数据、用户管理、全部单据操作
- `warehouse_keeper`：入/出/盘建单与审核
- `viewer`：可查库存与看板；可建改提交入/出库单；**不可**审核；盘点仅只读

默认种子账号：`admin` / `keeper` / `viewer`，密码均为 `admin123`。

---

### 2. `product` — 商品（编码）

| 字段 | 说明 |
|------|------|
| `sku_code` | 编码，唯一（如 `SKU-P001`） |
| `name` / `spec` / `unit` | 名称、规格、单位 |
| `barcode` | 条码（可选） |
| `safe_stock` | 安全库存，低于此值触发预警 |
| `is_active` | 是否启用 |

所有库存、单据明细都通过 `product_id` 关联到本表。

---

### 3. `warehouse` — 仓库

| 字段 | 说明 |
|------|------|
| `code` | 仓库编码，唯一（种子：`WH01`） |
| `name` | 仓库名称 |
| `address` | 地址（可选） |
| `is_active` | 是否启用 |

一个仓库下有多个库位；入库/出库/盘点单都指定目标仓库。

---

### 4. `location` — 库位

| 字段 | 说明 |
|------|------|
| `warehouse_id` | 所属仓库 |
| `code` | 库位编码，**同仓库内唯一** |
| `name` | 库位名称（可选） |
| `is_active` | 是否启用 |

约束：`UNIQUE (warehouse_id, code)`  
库存精确到「哪个仓库的哪个库位」，例如 `A-01-01`。

---

## 库存

### 5. `inventory` — 现存量

| 字段 | 说明 |
|------|------|
| `product_id` | 商品 |
| `warehouse_id` | 仓库 |
| `location_id` | 库位 |
| `quantity` | 当前数量（`DECIMAL(18,3)`） |
| `updated_at` | 更新时间 |

约束：`UNIQUE (product_id, warehouse_id, location_id)` — 每个「编码 + 仓库 + 库位」只有一条记录。

**注意**：应用代码禁止直接 UPDATE 此表，必须走 `inventory_service`。

---

### 6. `inventory_transaction` — 库存流水

| 字段 | 说明 |
|------|------|
| `product_id` / `warehouse_id` / `location_id` | 变动对象 |
| `delta_qty` | 变动量（正=入库/盘盈，负=出库/盘亏） |
| `balance_after` | 变动后余额 |
| `ref_type` | 来源类型：`inbound` / `outbound` / `stocktake` / `adjust` |
| `ref_id` | 关联单据 ID |
| `operator_id` | 操作人 |
| `remark` | 备注 |
| `created_at` | 时间 |

用于审计、对账、追溯。每次改 `inventory.quantity` 时同步插入一条流水。

---

## 入库

### 7. `inbound_order` — 入库单主表

| 字段 | 说明 |
|------|------|
| `order_no` | 单号，唯一，如 `IN202607220001` |
| `warehouse_id` | 入库目标仓库 |
| `supplier_name` | 供应商（字符串，可后续拆表） |
| `status` | 单据状态（见文末） |
| `remark` | 备注 |
| `created_by` / `approved_by` | 创建人、审核人 |
| `approved_at` / `completed_at` | 审核、完成时间 |

**状态流转**：`draft` → `submitted` → 审核过账后直接 `completed`（或 `cancelled`）；驳回回到 `draft`。  
审核通过时：按明细数量**增加**库存，并写流水。

---

### 8. `inbound_order_line` — 入库单明细

| 字段 | 说明 |
|------|------|
| `inbound_order_id` | 所属入库单 |
| `product_id` | 商品 |
| `location_id` | 入库目标库位 |
| `planned_qty` | 计划数量 |
| `actual_qty` | 实收数量（审核时以 actual 为准，为 0 则用 planned） |

删除主单时明细级联删除（`ON DELETE CASCADE`）。

---

## 出库

### 9. `outbound_order` — 出库单主表

| 字段 | 说明 |
|------|------|
| `order_no` | 单号，如 `OUT202607220001` |
| `warehouse_id` | 出库仓库 |
| `customer_name` | 客户（字符串） |
| `status` | 同入库单状态机 |
| `created_by` / `approved_by` 等 | 同入库 |

审核通过时：校验可用库存，不足则拒绝（`409 INSUFFICIENT_STOCK`）；足够则**扣减**库存并写流水。

---

### 10. `outbound_order_line` — 出库单明细

| 字段 | 说明 |
|------|------|
| `outbound_order_id` | 所属出库单 |
| `product_id` / `location_id` | 从哪个库位出哪个编码 |
| `planned_qty` / `actual_qty` | 计划 / 实际出库数量 |

---

## 盘点

### 11. `stocktake_order` — 盘点单主表

| 字段 | 说明 |
|------|------|
| `order_no` | 单号，如 `ST202607220001` |
| `warehouse_id` | 盘点范围仓库 |
| `status` | 同其他单据状态机 |

审核过账时：快照账面数量，按明细差异调整库存（盘盈加、盘亏减）。  
规则：`diff ≠ 0` 才调整；`counted_qty = 0` 且账面 `> 0` 时拒绝过账。

---

### 12. `stocktake_order_line` — 盘点单明细

| 字段 | 说明 |
|------|------|
| `stocktake_order_id` | 所属盘点单 |
| `product_id` / `location_id` | 盘点对象 |
| `book_qty` | 账面数量（过账时从 `inventory` 快照） |
| `counted_qty` | 实盘数量（录入） |
| `diff_qty` | 差异 = `counted_qty - book_qty` |

---

## 辅助

### 13. `order_no_seq` — 单据号序号

| 字段 | 说明 |
|------|------|
| `prefix` | 前缀：`IN` / `OUT` / `ST` |
| `seq_date` | 日期 |
| `last_seq` | 当日已用到第几号 |

约束：`UNIQUE (prefix, seq_date)`  
配合 `WmsServer/app/utils/order_no.py` 生成不重复单号，格式：`前缀 + YYYYMMDD + 4 位序号`。

---

## 单据状态说明（三张主单通用）

| 状态 | 含义 | 典型操作 |
|------|------|----------|
| `draft` | 草稿 | 编辑、提交、删除 |
| `submitted` | 已提交待审 | 审核通过、驳回、取消 |
| `approved` | 枚举保留；实现中不持久停留 | 审核瞬间可能短暂写入后立即完成 |
| `completed` | 已完成，库存已过账 | 只读 |
| `cancelled` | 已取消 | 只读 |

对外主流程：`draft` → `submitted` → `completed` / `cancelled`；驳回：`submitted` → `draft`。

---

## 如何使用本目录

1. **Navicat / 命令行**：执行 `init.sql` 创建库表结构  
2. **可选**：执行 `seed_users.sql` 仅写入 3 个用户  
3. **推荐应用初始化**：`cd WmsServer && python init_db.py` 写入完整种子数据  
4. **改表时**：同步修改 `init.sql` → `WmsServer/app/models/` → 必要时 Flask-Migrate 迁移
