# 香橙派部署指南（Orange Pi）

将本 WMS 部署到香橙派（以 **Orange Pi Zero 2W** 为参考）。  
本地开发请看 [README.md](./README.md)。

**推荐架构：** MySQL 放另一台机器/NAS；板子只跑后端（+ 可选 Nginx 托管前端）。板子内存紧张时不要在板上执行 `npm run build`，在 PC 上构建后拷贝 `dist`。

```
浏览器 ──▶ Nginx（板子，静态 dist）── /api 反代 ──▶ Waitress :5000 ──▶ MySQL（本机或远程）
```

可选：审核通过后 GPIO 脉冲（物理引脚 7 / PC9）。

---

## 1. 环境要求

| 组件 | 说明 |
|------|------|
| 系统 | Debian / Ubuntu 系镜像 |
| Python | 3.10+ |
| Node.js | 仅在 **PC 上构建前端** 时需要；板子可不装 |
| MySQL | 8.0+ 或 MariaDB；**优先远程**，本机可选 |
| GPIO | `gpiod`（提供 `gpioset`），需要引脚控制时安装 |

## 2. 获取代码

```bash
git clone https://github.com/vvb5456/WMS.git
cd WMS
```

---

## 3. 数据库

在 MySQL 所在机器执行：

```bash
mysql -u root -p < WmsSQL/init.sql
```

若库在远程，确保板子 IP 被允许连接，并记下主机、端口、账号密码。

---

## 4. 后端

```bash
cd WmsServer
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

编辑 `.env`：

| 变量 | 说明 |
|------|------|
| `SECRET_KEY` | 换成随机长串 |
| `MYSQL_HOST` 等 | 指向实际数据库（远程则填远程 IP） |
| `GPIO_ENABLED` | 需要审核脉冲时设 `true`，否则 `false` |
| `GPIO_CHIP` / `GPIO_LINE` / `GPIO_PULSE_SECONDS` | 默认 `gpiochip1` / `73` / `5` |

初始化数据并试跑：

```bash
python init_db.py
python serve.py
# 监听 0.0.0.0:5000
```

默认账号：`admin` / `keeper` / `viewer`，密码 `admin123`。**部署后立即改密。**

健康检查：

```bash
curl http://127.0.0.1:5000/api/v1/health
```

### systemd 常驻（推荐）

创建 `/etc/systemd/system/wms.service`（路径按实际修改）：

```ini
[Unit]
Description=WMS Flask Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/path/to/WMS/WmsServer
EnvironmentFile=/path/to/WMS/WmsServer/.env
ExecStart=/path/to/WMS/WmsServer/.venv/bin/python serve.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> `User`：若开 GPIO，需能访问 `/dev/gpiochip*`；权限不够时可先用能访问 GPIO 的用户，或配置 udev/组权限。

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now wms
sudo systemctl status wms
```

## 5. 前端

在 **PC** 上构建（推荐）：

```bash
cd WmsClient
cp .env.example .env
# 保持 VITE_API_BASE_URL=/api/v1（由板子 Nginx 反代）
npm install
npm run build
```

把 `WmsClient/dist/` 拷到板子，例如 `/var/www/wms/`。

板子安装 Nginx：

```bash
sudo apt update
sudo apt install -y nginx
```

配置示例（`/etc/nginx/sites-available/wms`）：

```nginx
server {
    listen 80;
    server_name _;

    root /var/www/wms;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

```bash
sudo ln -sf /etc/nginx/sites-available/wms /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

浏览器访问：`http://<香橙派IP>/`。

## 6. GPIO（可选）

安装工具：

```bash
sudo apt install -y gpiod
```

`.env` 示例：

```env
GPIO_ENABLED=true
GPIO_CHIP=gpiochip1
GPIO_LINE=73
GPIO_PULSE_SECONDS=5
```

说明：物理引脚 **7** ≈ PC9 ≈ `gpiochip1` line **73**（以你板子原理图为准，不对就改 chip/line）。审核通过后异步拉高约 N 秒；失败只记日志，不阻断审核。

手动试脉冲（确认线号无误后再测）：

```bash
gpioset --mode=time --sec=1 gpiochip1 73=1
```

未接线或不需要时：`GPIO_ENABLED=false`。

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| 页面刷新 404 | Nginx 加 `try_files ... /index.html` |
| 接口 404 / 登录失败 | 查 `/api` 反代与后端是否在跑 |
| 看板数据与所选日期对不上 | 确认后端已部署含东八区换算的 `product-movement`；流水时间为 UTC |
| 连不上库 | 查 `MYSQL_HOST`、远程 MySQL 授权与防火墙 |
| 审核无电平 | 查 `GPIO_ENABLED`、chip/line、用户对 `/dev/gpiochip*` 的权限、`gpioset` 是否安装 |
| 内存不足 | MySQL 改远程；前端在 PC 上 build；不要在板上跑 `npm run build` |
