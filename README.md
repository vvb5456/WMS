# 仓库管理系统 (WMS)

前后端分离的仓库管理系统：Vue 3 + Element Plus + Flask + MySQL。

库存变更**仅**通过后端 `inventory_service`，每次变动写入流水；入/出库/盘点走「草稿 → 提交 → 审核过账 → 完成」。

## 已实现功能

- 登录鉴权（JWT）、改密、用户管理（admin）
- 商品 / 仓库 / 库位主数据
- 库存查询、流水、安全库存预警
- 入库单 / 出库单 / 盘点单（建单、提交、审核、驳回、取消）
- 首页看板：周期出入库汇总（按产品统计入/出库合计，支持日期与仓库筛选、查询、导出 CSV/Excel）
- 可选：审核通过后 GPIO 脉冲（Orange Pi，默认关闭）



## 功能选项卡简介

按左侧菜单实际能力整理（以当前前端为准）：


| 菜单        | 可见角色               | 实际能做什么                                                                                                    |
| --------- | ------------------ | --------------------------------------------------------------------------------------------------------- |
| **首页看板**  | 全部登录用户             | 三张统计：有库存的编码数、待审入库单数、安全库存预警条数；「周期出入库汇总」可选日期/仓库后点查询，按产品列出入出库合计与当前库存、安全库存，可导出 CSV/Excel；下方「库存预警」表列出低于安全库存的明细 |
| **产品管理**  | 仅 admin            | 商品列表（编码/名称/规格/单位/安全库存/启用状态）；新建、编辑、删除（硬删）。无搜索、无分页、无启停开关（状态仅展示）                                             |
| **仓库库位**  | 仅 admin            | 左侧选仓库，右侧看该仓库库位；可新建仓库、新建库位、删除库位。无编辑/删除仓库、无编辑库位                                                             |
| **库存查询**  | 全部登录用户             | 两个 Tab：「现存量」（仅数量>0）、「库存流水」。无筛选、无导出、无分页                                                                    |
| **入库单**   | admin / 仓管 / 查看者   | 列表看单号/仓库/状态/创建时间，点进详情；新建后点「提交」会创建并立刻提交（不单独存草稿）。草稿可提交或删除；待审时仓管/admin 可审核通过、驳回、取消。列表无筛选                     |
| **出库单**   | admin / 仓管 / 查看者   | 流程同入库；草稿多「保存」；明细从有库存的库位选品，数量不超过可用量。列表无筛选                                                                  |
| **盘点单**   | admin / 仓管（查看者不可见） | 新建可「按库存生成明细」后改实盘并保存为草稿；可提交、删除；待审可审核过账/驳回/取消。不能手工加行，只能生成后删行                                                |
| **增加新用户** | 仅 admin            | 用户列表；增加用户（默认密码 `888888`）、重置密码为 `888888`、删除。无编辑资料、无改角色、无启停开关                                               |


顶栏：**修改密码**（成功后需重新登录）、**退出**。

## 本地开发



### 1. 数据库

启动本地 MySQL，创建库并执行初始化脚本：

```bash
mysql -u root -p < WmsSQL/init.sql
```

连接参数见 `WmsServer/.env.example`（默认库名 `wmsdatabase`，密码 `wmspassword`）。

仅需用户种子时可另执行 `WmsSQL/seed_users.sql`；完整示例数据推荐用下方 `init_db.py`。

### 2. 后端

```bash
cd WmsServer
pip install -r requirements.txt
cp .env.example .env
python init_db.py
python serve.py
```

- 健康检查：[http://localhost:5000/api/v1/health](http://localhost:5000/api/v1/health)
- `init_db.py`：建表 + 种子（用户、仓库 WH01、库位、商品 SKU-P001/P002）



### 3. 前端

```bash
cd WmsClient
npm install
npm run dev
```

访问 http://localhost:5173（Vite 代理 `/api` → `localhost:5000`）
